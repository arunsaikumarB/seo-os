# SEO OS — Complete Beginner’s Guide

**Who this is for:** Anyone new to SEO OS — freshers, interns, marketers, or founders who need to understand *what this product is*, *what each tool does*, and *how to use the full process*.

**What SEO OS is in one sentence:**  
SEO OS is an **AI-powered SEO operations platform** that helps a team discover website opportunities, plan campaigns, build backlinks, send outreach, track results, and keep the site technically healthy — all from one place.

<a id="1-big-picture"></a>
## 1. Big picture — why SEO OS exists

Traditional SEO work is scattered:

- One tool for site audits  
- Another for keywords  
- Spreadsheets for prospects  
- Gmail for outreach  
- Another dashboard for reports  

**SEO OS connects that work into one workflow**, with AI helping at each step:

```
Understand your site
        ↓
Find opportunities (pages/sites that can give you links or traffic)
        ↓
Plan campaigns
        ↓
Write & send outreach (with human approval)
        ↓
Verify backlinks & measure results
        ↓
Fix technical SEO issues & report to leadership
```

You do **not** need to be an SEO expert to use the basics. Follow the process below; AI and guided tips fill the gaps.

<a id="2-core-concepts"></a>
## 2. Core concepts (must know)

### Organization
Your **company or agency**. Users, billing (future), team roles, and company-wide settings live here.

### Project
One **website / client site** you are optimizing (example: `chefgaa.com`). Almost all SEO work happens inside a Project.

> Rule of thumb: **1 Project = 1 website**.

### Mission Control
The **operations dashboard** for a Project — live status of AI, campaigns, outreach, workflows, health, integrations, etc.

### Opportunity
A **chance to earn a backlink or SEO win** (guest post site, broken link, directory, resource page, etc.).

### Campaign
A **planned batch of work** built from opportunities (targets, content, outreach, approvals).

### Backlink
A **link from another website to yours**. Good backlinks help search visibility and authority.

### Workflow
An **automated sequence** (when X happens → do Y → ask for approval → notify).

### Demo Mode
A safe **sample environment** with fake-but-realistic data so you can learn without breaking a real client project.

## 3. How to start (first 15 minutes)

### Step A — Create an account
1. Open the app → **Sign up** / **Log in**.  
2. Complete your profile if asked.

### Step B — Create Organization
1. Go through **Onboarding** → create your Organization (company name).  
2. Invite teammates later from **Team**.

### Step C — Create a Project
1. Create a Project for the website you care about.  
2. Enter the site name and domain/URL.

### Step D — Open Mission Control
1. Open the Project.  
2. Go to **Mission Control**.  
3. Optionally start the **guided product tour** (Help → Restart guided tour).

### Step E — Run your first useful actions
1. **Browser Intelligence / Website Analyzer** — scan the site.  
2. **Knowledge Base** — upload a short brand/about PDF or notes.  
3. **Backlink Builder / Opportunity Explorer** — see opportunities.  

If you only do those three, you already understand 70% of the product.

<a id="4-the-recommended-process"></a>
## 4. The recommended end-to-end process

This is the **standard SEO OS loop**. Use it every time for a new site.

| # | Step | Where in the app | What you do | What you get |
|---|------|------------------|-------------|--------------|
| 1 | Create Organization | Onboarding / Org | Name your company | Team workspace |
| 2 | Create Project | Onboarding / Projects | Add the website | Project home |
| 3 | Analyze website | Browser Intelligence / Website Analyzer | Run a scan | Pages, contacts, tech signals |
| 4 | Teach the AI | Knowledge Base + AI Memory | Upload docs / approve facts | Better AI writing & strategy |
| 5 | Discover opportunities | Backlink Builder / Explorer / Prospects | Browse & filter opportunities | List of link targets |
| 6 | Plan campaigns | Campaigns / Opportunity Queue | Select opportunities → create campaign | Structured plan |
| 7 | Approve content & emails | Content / Outreach Studio | Generate drafts | Emails, pitches, articles |
| 8 | Approve (human check) | Approval Center | Approve or reject AI output | Safe-to-send content |
| 9 | Send outreach | Outreach Inbox / Sequences | Send or schedule | Conversations with prospects |
| 10 | Automate repeats | Workflows | Turn on templates | Less manual follow-up |
| 11 | Verify wins | Backlink Builder → Pending / Won | Check if links went live | Verified backlinks |
| 12 | Measure | Analytics + Reports | Review KPIs / export report | Client/leadership updates |
| 13 | Fix site health | Technical SEO | Run audit → fix issues | Health score improves |
| 14 | Connect tools | Integrations | Connect GSC, GA4, Slack, email | Live external data |
| 15 | Monitor | Mission Control | Check widgets daily | One-screen operations |

**Mental model:**  
*Discover → Plan → Create → Approve → Send → Verify → Report → Improve.*

<a id="5-every-tool-explained"></a>
## 5. Every tool & category explained

Below: **What it is**, **Why it exists**, **How to use it**, and **When to use it**.

---

### 5.1 Mission Control
**What it is:** The live “control room” for your Project.  
**Why it exists:** So you don’t jump between 10 screens to see if work is healthy.  
**How to use:**
1. Open Project → **Mission Control**.  
2. Read widgets: AI workforce, campaigns, browser intelligence, outreach, workflows, analytics, reports, technical SEO, integrations.  
3. Click a widget’s “Open …” button to go deeper.  
**When:** Every morning, and after any major run (scan, campaign, audit).

---

### 5.2 Project Home / Overview
**What it is:** Simple project landing page.  
**Why:** Quick orientation for the site you’re working on.  
**How:** Project → **Home**.  
**When:** Switching between clients/sites.

---

### 5.3 Backlink Builder (Flagship)
**What it is:** The main system for finding and winning backlinks.  
**Why:** Backlinks are a major SEO growth lever; this module turns “link building” into a pipeline.  

**Key sub-areas:**
| Screen | Purpose |
|--------|---------|
| Dashboard | Pipeline overview (found → contacted → won/lost) |
| Opportunity Explorer | Search/browse opportunities |
| Opportunity detail | One opportunity’s score, type, next actions |
| Import | Bring websites/lists from CSV or external sources |
| AI Classification / Automation | AI labels & prioritizes opportunities |
| Recommendations | Suggested next best targets |
| Pipeline | Kanban-style stages |
| Pending verification | Links claimed but not confirmed |
| Won / Lost | Outcomes |
| Relationships (BB) | People/orgs tied to opportunities |
| Campaigns (BB) | Campaigns created from BB |
| Tracking | Track link status over time |
| Audit | Review link quality / issues |

**How to use (simple):**
1. Run website analysis first (so AI knows your niche).  
2. Open **Opportunity Explorer**.  
3. Filter high-score opportunities.  
4. Send promising ones to **Campaigns** or **Pipeline**.  
5. Later verify in **Pending → Won**.

**When:** Ongoing — this is the heart of growth work.

---

### 5.4 AI Command Center
**What it is:** A ChatGPT-like chat for your Project.  
**Why:** Ask questions or request actions in natural language (“summarize opportunities”, “draft a guest post pitch”).  
**How:** Project → **AI Command Center** → type a question.  
**When:** You want help without navigating every screen. Also used as the “AI Help Assistant” from Help Center.

---

### 5.5 AI Agents / AI Workforce
**What it is:** Specialized AI roles (strategist, researcher, outreach manager, QA, etc.).  
**Why:** Different tasks need different “experts.”  
**How:** Use from Mission Control / Command Center / agent runs. (Full Agents Catalog UI may still be limited; agents still run behind the scenes.)  
**When:** Campaign planning, research, content, QA checks.

---

### 5.6 Knowledge Base
**What it is:** Your library of brand documents (about us, services, tone of voice, case studies).  
**Why:** AI writes better when it knows *your* business — not generic internet text.  
**How:**
1. Project → **Knowledge Base**.  
2. Upload PDFs/docs or paste content.  
3. Wait until documents are processed/ready.  
**When:** Right after creating a Project, before generating outreach/content.

---

### 5.7 AI Memory
**What it is:** Facts the AI has learned about this Project (preferences, brand rules, past decisions).  
**Why:** So the AI stays consistent over weeks.  
**How:** Open **AI Memory** timeline → review/approve important facts.  
**When:** After Knowledge uploads and after major decisions (“we never pitch gambling sites”).

---

### 5.8 Website Analyzer
**What it is:** Higher-level website analysis for SEO intelligence.  
**Why:** Understand pages, topics, and site structure.  
**How:** Project → **Website Analyzer** → run/view analysis.  
**When:** Kickoff of a new Project.

---

### 5.9 Browser Intelligence
**What it is:** Deeper crawl/scan using browser-style intelligence (pages read, contacts, broken links, discoveries).  
**Why:** Powers opportunity discovery and technical signals.  
**How:** Project → **Browser Intelligence** → start a scan → open scan detail.  
**When:** Before discovery and before technical audits.

---

### 5.10 Competitors
**What it is:** Competitor tracking workspace.  
**Why:** See who you compete with for attention and links.  
**How:** Add/review competitors.  
**When:** Strategy phase and monthly reviews.

---

### 5.11 Keywords
**What it is:** Keyword research and tracking area.  
**Why:** Know what searches matter for the Project.  
**How:** Open **Keywords** → explore lists/signals.  
**When:** Content planning and opportunity scoring.

---

### 5.12 Prospects (Pipeline)
**What it is:** Potential partners/sites you may contact.  
**Why:** Turns raw discoveries into a contactable list.  
**How:** Move prospects through pipeline stages.  
**When:** Between discovery and outreach.

---

### 5.13 Campaigns
**What it is:** Planned SEO/backlink campaigns.  
**Why:** Stops random one-off emails; creates managed programs.  

**Related screens:**
| Screen | Purpose |
|--------|---------|
| Campaigns list | All campaigns |
| Campaign detail | Progress, timeline, tasks |
| Opportunity Queue | Opportunities waiting to become campaign work |
| Approval Center | Human approval gate for AI outputs |

**How:**
1. From opportunities, create or add to a Campaign.  
2. Let AI draft plan/content.  
3. Approve in **Approval Center**.  
4. Continue to Outreach.  

**When:** Weekly growth sprints.

---

### 5.14 Content Studio
**What it is:** Place for generated content assets (articles, pitches).  
**Status:** May appear as a placeholder depending on release — Outreach Studio covers much of draft generation today.  
**When:** Content-heavy campaigns.

---

### 5.15 Outreach Engine
**What it is:** Email drafting, inbox, and sequences for contacting prospects.  
**Why:** Winning links requires polite, personalized outreach.  

| Screen | Purpose |
|--------|---------|
| Inbox | Conversations / replies |
| Email Studio | Draft emails with AI |
| Sequences | Multi-step follow-ups |

**How:**
1. Generate draft in Studio.  
2. Get approval if required.  
3. Send (connected email provider via Integrations).  
4. Track replies in Inbox.  
5. Use Sequences for follow-ups.  

**When:** After campaign + approval.

---

### 5.16 Relationship Intelligence
**What it is:** CRM-like view of organizations and people you build relationships with.  
**Why:** Link building is relationship building — not only “send email once.”  
**How:** Open **Relationships** → browse organizations → open detail.  
**When:** Parallel to outreach and for long-term partners.

---

### 5.17 Workflow Automation
**What it is:** Visual/automation engine that runs multi-step processes.  
**Why:** Repeatable work (discover → draft → approve → notify) shouldn’t be fully manual.  

| Screen | Purpose |
|--------|---------|
| Workflows | Your automations |
| Templates | Ready-made recipes (guest post, broken link, critical SEO response, etc.) |
| Builder | Edit nodes (trigger → AI → approval → end) |
| Runs | History of executions |

**How:**
1. Open **Templates** → create from a template.  
2. Activate the workflow.  
3. Watch **Runs** when events fire.  

**When:** After you understand the manual process once.

---

### 5.18 Technical SEO Engine
**What it is:** Website health auditor (broken links, HTTPS, meta, schema, robots, sitemap, accessibility signals, etc.).  
**Why:** Great content/outreach fails if the site is technically sick.  

**How:**
1. Project → **Technical SEO**.  
2. Enter URL → choose **full / incremental / quick**.  
3. Start audit.  
4. Open issues → read severity, business/SEO impact, recommended fix.  
5. Use AI Fix Assistant suggestions.  
6. Mark fixed / ignored.  
7. Export CSV/JSON/PDF if needed.  

**When:** Monthly, and after big site changes.

---

### 5.19 Integrations Hub
**What it is:** Connect external tools to SEO OS.  

| Provider | Use |
|----------|-----|
| Google Search Console | Search queries, clicks, pages, devices |
| Google Analytics 4 | Sessions, users, conversions, sources |
| SMTP / Gmail / Outlook | Send outreach email |
| WordPress | Draft posts from AI (publishing stays user-controlled) |
| Slack | Alerts for workflows, approvals, critical SEO |

**How:**
1. Project **Integrations** or Org **Integrations**.  
2. Click **Connect** on a provider.  
3. Run **Sync**.  
4. Check health / permissions.  

**When:** After Project setup; before serious reporting.

---

### 5.20 Analytics
**What it is:** Measurement dashboards across SEO, campaigns, outreach, workflows, AI, etc.  
**Why:** Prove what is working.  
**How:** Open **Analytics** → switch sections (Executive, SEO, Backlinks, Campaigns…). Export if needed.  
**When:** Weekly reviews.

---

### 5.21 Reports
**What it is:** Client/leadership report generator (executive, campaign, backlink, monthly…).  
**Why:** Turn data into shareable PDF/PPTX/CSV/JSON.  
**How:**
1. **Reports** → create report type.  
2. Optional brand/white-label.  
3. Generate.  
4. Export or email delivery.  

**When:** Monthly client meetings or board updates.

---

### 5.22 Search (in-app)
**What it is:** Quick find across the Project.  
**How:** Use Search from mobile/nav or command palette (**Ctrl/⌘ + K**).  
**When:** You know the name but not the menu location.

<a id="7-tips-and-demo-mode"></a>
## 7. Tips, shortcuts, and Demo Mode

### Demo Mode
- Toggle Demo Mode from the user menu.  
- Explores realistic sample data safely.  
- Perfect for training freshers and client demos.

### Useful shortcuts
| Shortcut | Action |
|----------|--------|
| **Ctrl/⌘ + K** | Command palette (jump anywhere) |
| **Esc** | Close dialogs / tour |
| **Tab** | Move between fields/buttons |

### Freshers’ golden rules
1. Always work inside the correct **Project**.  
2. **Scan → Knowledge → Discover → Campaign → Approve → Outreach → Verify → Report**.  
3. Never send AI outreach without reading it (use Approvals).  
4. Check **Mission Control** before asking “is it working?”  
5. File bugs in **Feedback** instead of guessing.

<a id="8-glossary"></a>
## 8. Simple glossary

| Term | Meaning |
|------|---------|
| SEO | Search Engine Optimization — helping a site appear in search |
| Crawl / Scan | Automated visit of site pages to collect data |
| SERP | Search Engine Results Page |
| CTA | Call to action (button/link asking user to do something) |
| Domain authority (informal) | How “strong” a site is seen for links |
| Schema / JSON-LD | Structured data helping search engines understand pages |
| Canonical | Preferred URL when duplicates exist |
| Robots.txt | File telling crawlers what they may access |
| Sitemap | List of important URLs for search engines |
| CWV | Core Web Vitals — speed/UX quality signals |
| RBAC | Role-based access control (who can do what) |
